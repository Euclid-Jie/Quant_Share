数据表

目前的表多位ts组织形式，即时间轴t和个体标识s双重索引，其中date_cokumn与get_data中的begin，end区间一致，ticker_column与get_data中的ticker一致

需要说明的是，期货数据的组织形式与股票不同，每个期货品种的数据单独存储，故ticker直接指定期货品种代码即可，不设ticker_column

Info表

bench_info、stock_info、tradeDate_info已作为Utils中的常量，使用from Quant_share import *时，默认被导入

另有如下Info表可使用get_data(InfoName)直接导入

  • SecID_info：通联股票基本信息
  • IndustryID_info：通联行业行业分类标准
  • SysCode_info：通联参数常量值集合
  • PartyID_info：通联公司基本信息
  • TradeCal：通联沪深交易所交易日历
